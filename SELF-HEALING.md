## 错误分类

### API 错误
| 错误类型 | 原因 | 自愈策略 |
|----------|------|----------|
| 限流 | 请求过多 | 等待60秒后重试 |
| 超时 | 网络问题 | 重试3次，间隔递增 |
| 认证失败 | Token过期 | 提示用户更新 |
| 服务不可用 | 服务商问题 | 切换备用服务 |

### 工具错误
| 错误类型 | 原因 | 自愈策略 |
|----------|------|----------|
| 文件不存在 | 路径错误 | 创建文件或提示 |
| 权限不足 | 权限问题 | 提示用户授权 |
| 工具超时 | 处理过长 | 后台运行+通知 |
| 参数错误 | 参数不合法 | 修正参数重试 |

### 系统错误
| 错误类型 | 原因 | 自愈策略 |
|----------|------|----------|
| 内存不足 | 资源占用高 | 清理缓存 |
| 进程阻塞 | 长任务卡住 | 重启进程 |
| 会话丢失 | 重启导致 | 恢复任务 |

---

## 自愈流程

```
错误发生 → 分类错误 → 选择策略 → 执行自愈 → 记录日志

自愈失败 → 通知用户 → 提供方案 → 等待处理
```

---

## 重试机制

### 自动重试规则
| 错误类型 | 重试次数 | 间隔策略 |
|----------|----------|----------|
| 限流 | 3次 | 指数退避（60s, 120s, 240s）|
| 超时 | 3次 | 固定间隔（30s）|
| 网络错误 | 5次 | 指数退避（10s, 20s, 40s, 80s, 160s）|
| 服务不可用 | 1次 | 切换备用 |

### 重试日志
```json
{
  "error": "限流",
  "time": "2026-03-04 21:00:00",
  "retry": 1,
  "result": "成功"
}
```
