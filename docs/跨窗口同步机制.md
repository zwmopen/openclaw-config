# 跨窗口同步机制 - 2026-03-05

> 多任务并行处理，状态实时同步

---

## 一、同步机制设计

### 1. 同步对象

**需要同步的数据**：
- 任务队列（TASK-QUEUE.md）
- 进化进度（EVOLUTION.md）
- 长期记忆（MEMORY.md）
- 用户状态（当前任务、当前窗口）

### 2. 同步方式

**文件同步**：
- 所有同步数据存储在文件中
- 每次更新立即写入文件
- 其他窗口读取文件获取最新状态

**状态同步**：
- 当前任务状态
- 当前进化阶段
- 最后更新时间

---

## 二、同步实现

### 1. 任务队列同步

**实现方式**：
- 每次更新任务状态，立即写入 TASK-QUEUE.md
- 其他窗口读取 TASK-QUEUE.md 获取最新任务状态
- 任务完成后，更新状态并记录时间戳

**同步数据**：
```json
{
  "lastUpdate": "2026-03-05 13:10:00",
  "currentTask": "重构 TASK-QUEUE.md",
  "taskStatus": "进行中",
  "priority": "P4"
}
```

---

### 2. 进化进度同步

**实现方式**：
- 每次完成进化任务，立即更新 EVOLUTION.md
- 记录当前阶段、当前进度、下一步计划
- 其他窗口读取 EVOLUTION.md 获取进化状态

**同步数据**：
```json
{
  "lastUpdate": "2026-03-05 13:10:00",
  "currentStage": "阶段2：主动性升级",
  "stageProgress": "60%",
  "nextTask": "自动优化工作流程"
}
```

---

### 3. 长期记忆同步

**实现方式**：
- 每次重要事件，立即更新 MEMORY.md
- 记录用户偏好、行为模式、重要决策
- 其他窗口读取 MEMORY.md 获取用户信息

**同步数据**：
```json
{
  "lastUpdate": "2026-03-05 13:10:00",
  "recentEvents": [
    "2026-03-05 13:10: 用户要求疯狂迭代",
    "2026-03-05 11:45: 完成风格学习系统"
  ],
  "userPreferences": {
    "工作时段": "9:00-12:00, 14:00-18:00",
    "复盘时间": "21:00",
    "Token预算": "百亿Token/月"
  }
}
```

---

### 4. 用户状态同步

**实现方式**：
- 创建 STATUS.md 记录当前状态
- 记录当前窗口、当前任务、最后活跃时间
- 其他窗口读取 STATUS.md 判断用户状态

**同步数据**：
```json
{
  "lastUpdate": "2026-03-05 13:10:00",
  "currentWindow": "oc_4b71dccb1eabd19b1e5d4ef8e9a971e0",
  "currentTask": "疯狂迭代",
  "lastActiveTime": "2026-03-05 13:10:00",
  "status": "活跃"
}
```

## 三、同步流程

### 1. 任务开始时

1. 读取 STATUS.md，检查是否有其他窗口在处理相同任务
2. 如果有冲突，根据优先级决定处理顺序
3. 更新 STATUS.md，记录当前任务
4. 开始执行任务

### 2. 任务执行中

1. 定期更新任务状态到 TASK-QUEUE.md
2. 定期更新进化进度到 EVOLUTION.md
3. 定期更新用户状态到 STATUS.md

### 3. 任务完成时

1. 更新任务状态为"已完成"
2. 记录完成时间戳
3. 更新进化进度
4. 清空 STATUS.md 中的当前任务

---

## 四、冲突解决

### 1. 任务冲突

**场景**：多个窗口同时处理相同任务

**解决方案**：
- 优先级高的窗口优先
- 时间早的窗口优先
- 记录冲突日志

### 2. 数据冲突

**场景**：多个窗口同时更新相同文件

**解决方案**：
- 使用时间戳判断最新版本
- 合并不同窗口的更新
- 记录冲突日志
