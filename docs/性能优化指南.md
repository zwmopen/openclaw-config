# OpenClaw 性能优化指南

## 问题分析

### 1. 模型响应时间
- 联通元景GLM-5: ~1624ms
- 硅基流动DeepSeek-V3: ~1394ms
- 智谱GLM-4-Flash: 可能更快

### 2. 上下文管理
- 当前上下文窗口: 128000 tokens
- 上下文修剪模式: cache-ttl
- 会话记忆: 已启用

### 3. 文件读取
- 工作目录文件数量
- 每次请求可能读取多个文件
- 缺少向量化检索

---

## 优化方案

### 方案1: 模型切换（最简单）

切换到更快的模型：

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "custom-api-siliconflow-cn/deepseek-ai/DeepSeek-V3"
      }
    }
  }
}
```

**优点**: 立即生效，响应时间减少约15%
**缺点**: 需要切换模型提供商

---

### 方案2: 上下文优化（推荐）

优化上下文修剪策略：

```json
{
  "agents": {
    "defaults": {
      "contextPruning": {
        "mode": "cache-ttl",
        "ttl": "30m",
        "keepLastAssistants": 2,
        "softTrimRatio": 0.5,
        "hardClearRatio": 0.7
      },
      "compaction": {
        "mode": "aggressive",
        "reserveTokens": 8192,
        "keepRecentTokens": 10000
      }
    }
  }
}
```

**优点**: 减少上下文长度，加快处理速度
**缺点**: 可能丢失一些历史信息

---

### 方案3: 向量化/RAG配置（高级）

配置向量数据库，实现智能检索：

```json
{
  "agents": {
    "defaults": {
      "rag": {
        "enabled": true,
        "provider": "chromadb",
        "embeddingModel": "text-embedding-3-small",
        "chunkSize": 1000,
        "chunkOverlap": 200,
        "topK": 5
      }
    }
  }
}
```

**优点**: 只检索相关内容，大幅减少上下文
**缺点**: 需要额外配置向量数据库

---

### 方案4: 缓存策略优化

启用响应缓存：

```json
{
  "agents": {
    "defaults": {
      "cache": {
        "enabled": true,
        "ttl": "1h",
        "similarityThreshold": 0.95
      }
    }
  }
}
```

**优点**: 相同问题直接返回缓存结果
**缺点**: 需要额外内存
