## 🔍 影响API调用次数的机制

### 1. 心跳检查
- **当前设置**：每5分钟检查一次
- **API调用**：0次（只回HEARTBEAT_OK，不发消息）
- **优化空间**：可以更频繁（流畅模式）或更少（保守模式）

### 2. 定时提醒
- **当前设置**：每天约5-6次
- **API调用**：5-6次/天
- **提醒时间**：
  - 8:00 早间问候
  - 12:00 进化进度汇报
  - 21:00 深度复盘
  - 23:00 清空滴答清单

### 3. 用户消息交互
- **估算**：每天2-10次对话
- **API调用**：2-10次/天
- **优化空间**：
  - 流畅模式：可以多发消息，详细回复
  - 保守模式：精简回复，合并多条消息

### 4. 工具调用结果
- **当前行为**：每次工具调用后返回结果
- **API调用**：每次工具调用 = 1次API调用
- **优化空间**：
  - 流畅模式：显示所有工具调用过程
  - 保守模式：隐藏工具调用，只显示最终结果

### 5. 实时状态展示
- **潜在功能**：实时显示AI状态、思考过程
- **API调用**：每次状态更新 = 1次API调用
- **优化空间**：
  - 流畅模式：实时显示状态
  - 保守模式：关闭实时状态
