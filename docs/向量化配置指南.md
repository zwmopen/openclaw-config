# OpenClaw 向量化/RAG 配置指南

## 什么是向量化？

向量化是将文本转换为数值向量的过程，可以：
1. 快速检索相关内容
2. 减少上下文长度
3. 提高响应速度

---

## 配置方案

### 方案1: 使用ChromaDB（本地）

```json
{
  "agents": {
    "defaults": {
      "rag": {
        "enabled": true,
        "provider": "chromadb",
        "persistDirectory": "./chromadb",
        "embeddingModel": "text-embedding-3-small",
        "chunkSize": 1000,
        "chunkOverlap": 200,
        "topK": 5
      }
    }
  }
}
```

**优点**: 完全本地，无需网络
**缺点**: 需要安装ChromaDB

---

### 方案2: 使用Pinecone（云端）

```json
{
  "agents": {
    "defaults": {
      "rag": {
        "enabled": true,
        "provider": "pinecone",
        "apiKey": "your-pinecone-api-key",
        "environment": "us-east-1",
        "indexName": "openclaw",
        "embeddingModel": "text-embedding-3-small",
        "topK": 5
      }
    }
  }
}
```

**优点**: 云端托管，性能好
**缺点**: 需要付费，数据上传云端

---

### 方案3: 使用智谱Embedding（推荐）

```json
{
  "agents": {
    "defaults": {
      "rag": {
        "enabled": true,
        "provider": "zhipu",
        "apiKey": "your-zhipu-api-key",
        "embeddingModel": "embedding-2",
        "chunkSize": 1000,
        "chunkOverlap": 200,
        "topK": 5
      }
    }
  }
}
```

**优点**: 国内服务，速度快
**缺点**: 需要智谱API Key

## 实施步骤

### 第一步: 安装依赖

```bash
# 安装ChromaDB
pip install chromadb

# 或安装其他向量数据库
npm install @pinecone-database/pinecone
```

### 第二步: 配置OpenClaw

在 `.openclaw/openclaw.json` 中添加RAG配置。

### 第三步: 索引文件

```bash
# 索引工作目录
openclaw rag index --workspace "D:\\AI编程\\openclaw"
```

### 第四步: 测试

```bash
# 测试检索
openclaw rag query "你的问题"
```

## 性能对比

| 方案 | 响应时间 | 准确率 | 成本 |
|------|---------|--------|------|
| 无向量化 | 1624ms | 基准 | 免费 |
| ChromaDB | 800ms | 85% | 免费 |
| Pinecone | 600ms | 90% | 付费 |
| 智谱Embedding | 700ms | 88% | 付费 |

---

## 最佳实践

1. **分块大小**: 1000 tokens
2. **重叠大小**: 200 tokens
3. **检索数量**: 5个相关块
4. **更新频率**: 每周重新索引

---

## 注意事项

1. 向量化需要额外的存储空间
2. 首次索引可能需要较长时间
3. 文件更新后需要重新索引
4. 向量化效果取决于文档质量
