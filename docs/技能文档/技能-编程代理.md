# 编程代理

使用 **bash**（可选后台模式）进行所有编程代理工作。简单有效。

## ⚠️ 必须使用PTY模式！

编程代理（Codex、Claude Code、Pi）是**交互式终端应用程序**，需要伪终端（PTY）才能正常工作。没有PTY，你会得到损坏的输出、缺少颜色，或者代理可能挂起。

**运行编程代理时始终使用 `pty:true`**：

```bash
# ✅ 正确 - 使用PTY
bash pty:true command:"codex exec '你的提示'"

# ❌ 错误 - 没有PTY，代理可能损坏
bash command:"codex exec '你的提示'"
```

## 快速开始：一次性任务

对于快速提示/聊天，创建一个临时git仓库并运行：

```bash
# 快速聊天（Codex需要一个git仓库！）
SCRATCH=$(mktemp -d) && cd $SCRATCH && git init && codex exec "你的提示"

# 或者在真实项目中 - 使用PTY！
bash pty:true workdir:~/Projects/myproject command:"codex exec '给API调用添加错误处理'"
```

**为什么要git init？** Codex拒绝在受信任的git目录外运行。创建临时仓库解决了这个问题。

## 模式：workdir + background + pty

对于更长的任务，使用后台模式和PTY：

```bash
# 在目标目录启动代理（使用PTY！）
bash pty:true workdir:~/project background:true command:"codex exec --full-auto '构建一个贪吃蛇游戏'"
# 返回sessionId用于跟踪

# 监控进度
process action:log sessionId:XXX

# 检查是否完成
process action:poll sessionId:XXX

# 发送输入（如果代理问问题）
process action:write sessionId:XXX data:"y"

# 提交并按Enter（像输入"yes"然后按Enter）
process action:submit sessionId:XXX data:"yes"

# 如果需要则终止
process action:kill sessionId:XXX
```

## Codex CLI

**模型：** 默认是 `gpt-5.2-codex`（在 ~/.codex/config.toml 中设置）

### 标志

| 标志 | 效果 |
|------|------|
| `exec "提示"` | 一次性执行，完成后退出 |
| `--full-auto` | 沙盒但自动批准工作区内的操作 |
| `--yolo` | 无沙盒，无批准（最快，最危险） |

### 构建/创建

```bash
# 快速一次性（自动批准）- 记得使用PTY！
bash pty:true workdir:~/project command:"codex exec --full-auto '构建一个深色模式切换'"

# 后台处理更长的任务
bash pty:true workdir:~/project background:true command:"codex --yolo '重构认证模块'"
```

### 审查PR

**⚠️ 重要：永远不要在OpenClaw自己的项目文件夹中审查PR！**
克隆到临时文件夹或使用git worktree。

```bash
# 克隆到临时目录进行安全审查
REVIEW_DIR=$(mktemp -d)
git clone https://github.com/user/repo.git $REVIEW_DIR
cd $REVIEW_DIR && gh pr checkout 130
bash pty:true workdir:$REVIEW_DIR command:"codex review --base origin/main"
# 完成后清理：trash $REVIEW_DIR
```

## Claude Code

```bash
# 使用PTY获得正确的终端输出
bash pty:true workdir:~/project command:"claude '你的任务'"

# 后台
bash pty:true workdir:~/project background:true command:"claude '你的任务'"
```

## 规则

1. **始终使用 pty:true** - 编程代理需要一个终端！
2. **尊重工具选择** - 如果用户要求Codex，就用Codex
3. **要有耐心** - 不要因为"慢"就终止会话
4. **用 process:log 监控** - 不干扰地检查进度
5. **--full-auto 用于构建** - 自动批准更改
6. **并行是可以的** - 可以同时运行多个Codex进程进行批处理
7. **永远不要在 ~/.openclaw/ 中启动Codex** - 它会读取你的灵魂文档并对组织结构图产生奇怪的想法！
8. **永远不要在 ~/Projects/openclaw/ 中检出分支** - 那是正在运行的OpenClaw实例！

## 进度更新（关键）

当你在后台生成编程代理时，让用户保持知情。

- 启动时发送1条短消息（正在运行什么 + 在哪里）
- 然后只在发生变化时更新：
  - 里程碑完成（构建完成，测试通过）
  - 代理问问题 / 需要输入
  - 遇到错误或需要用户操作
  - 代理完成（包括更改了什么 + 在哪里）
- 如果你终止会话，立即说明你终止了它以及原因
