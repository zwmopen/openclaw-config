# 心跳模式切换技能

> **触发词**：高频模式、低频模式、切回高频、切回低频、恢复高频、恢复低频

---

## 模式说明

### 高频模式

**用途**：飞书API无限额度时使用，实时展示状态

**配置**：
- 心跳间隔：35秒
- 定时提醒：全部开启
- 进度汇报：实时发送

**定时提醒**：
| 时间 | 提醒内容 | 状态 |
|------|---------|------|
| 每天 8:00 | 早间问候 | ✅ |
| 工作时间每30分钟 | 👀 眼睛休息提醒 | ✅ |
| 工作时间每2小时 | 💧 喝水提醒 | ✅ |
| 工作时间每45分钟 | 🍅 番茄钟提醒 | ✅ |
| 每天 21:00 | 深度认知复盘 | ✅ |
| 每天 23:00 | 提醒清空滴答清单 | ✅ |
| 每小时 | 进度汇报 | ✅ |

### 低频模式

**用途**：飞书API限制时使用，节省消息额度

**配置**：
- 心跳间隔：60分钟
- 定时提醒：只保留早间问候和深度复盘
- 进度汇报：静默模式（只记录，不发消息）

**定时提醒**：
| 时间 | 提醒内容 | 状态 |
|------|---------|------|
| 每天 8:00 | 早间问候 | ✅ 保留 |
| 工作时间每30分钟 | 👀 眼睛休息提醒 | ⏸️ 暂停 |
| 工作时间每2小时 | 💧 喝水提醒 | ⏸️ 暂停 |
| 工作时间每45分钟 | 🍅 番茄钟提醒 | ⏸️ 暂停 |
| 每天 21:00 | 深度认知复盘 | ✅ 保留 |
| 每天 23:00 | 提醒清空滴答清单 | ⏸️ 暂停 |
| 每小时 | 进度汇报 | ✅ 保留（静默） |

## 切换命令

### 切换到高频模式

**用户说**：
- `高频模式`
- `切回高频`
- `恢复高频`
- `飞书API无限额度了`
- `申请到无限额度了`

**OpenClaw执行**：
1. 修改 `HEARTBEAT.md`：
   - 心跳间隔改回35秒
   - 定时提醒全部开启
   - 进度汇报改回实时模式
2. 修改 `MEMORY.md`：
   - 删除"飞书API限制"相关的临时配置
3. 删除 `docs/高频发消息配置备份.md`（如果存在）
4. 回复：`✅ 高频模式已恢复！心跳35秒，定时提醒全部开启，进度汇报实时发送。`

### 切换到低频模式

**用户说**：
- `低频模式`
- `切回低频`
- `恢复低频`
- `飞书API限制了`
- `额度快用完了`

**OpenClaw执行**：
1. 创建 `docs/高频发消息配置备份.md`（备份当前配置）
2. 修改 `HEARTBEAT.md`：
   - 心跳间隔改为60分钟
   - 定时提醒只保留早间问候和深度复盘
   - 进度汇报改为静默模式
3. 修改 `MEMORY.md`：
   - 添加"飞书API限制"相关的临时配置
4. 回复：`✅ 低频模式已启动！心跳60分钟，只保留早间问候（8:00）和深度复盘（21:00），其他提醒暂停。`

## 配置文件

### HEARTBEAT.md 配置

**高频模式（开头）**：
```markdown
# HEARTBEAT.md

> **高频模式**：已恢复！飞书API无限额度
> **心跳间隔**：35秒
> **定时提醒**：全部开启
```

**低频模式（开头）**：
```markdown
# HEARTBEAT.md

> **临时配置**：飞书API限制，心跳改为60分钟，减少消息
> **备份配置**：见 `docs/高频发消息配置备份.md`
> **提醒**：申请无限额度后切回高频模式
```

### MEMORY.md 配置

**低频模式时添加**：
```markdown
## 飞书API限制（重要！）

**限制**：每月1万条消息
**当前模式**：低频模式（心跳60分钟）
**配置备份**：`docs/高频发消息配置备份.md`
**提醒**：申请无限额度后切回高频模式

### 临时配置（生效中）
- 心跳间隔：60分钟（原35秒）
- 每小时汇报一次进度
- 定时提醒：只保留早间问候（8:00）和深度复盘（21:00）
- 其他提醒：暂停
```

**高频模式时删除此段**

## 使用场景

### 场景1：刚申请到无限额度

**用户说**：`申请到无限额度了，切回高频模式`

**OpenClaw执行**：
1. 切换到高频模式
2. 告诉用户：`✅ 高频模式已恢复！现在可以尽情发消息了！`

### 场景2：额度快用完了

**用户说**：`额度快用完了，切回低频模式`

**OpenClaw执行**：
1. 备份当前配置
2. 切换到低频模式
3. 告诉用户：`✅ 低频模式已启动！额度用完前会尽量节省消息。`

### 场景3：不知道当前模式

**用户说**：`当前是什么模式？`

**OpenClaw回复**：
```
当前是【高频模式】：
- 心跳间隔：35秒
- 定时提醒：全部开启
- 进度汇报：实时发送
```

或

```
当前是【低频模式】：
- 心跳间隔：60分钟
- 定时提醒：只保留早间问候（8:00）和深度复盘（21:00）
- 进度汇报：静默模式
```
