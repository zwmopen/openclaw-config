# Trae Space Cleaner

This skill helps analyze and clean Trae CN disk space usage.

## When to Invoke

- User mentions "空间不够" (not enough space)
- User mentions "磁盘满了" (disk full)
- User asks about "Trae 占用空间" (Trae space usage)
- User wants to clean or optimize Trae storage
- User mentions "清理" (clean) in context of Trae

## Trae CN Storage Locations

| Location | Path | Description |
|----------|------|-------------|
| Roaming | `c:\Users\{user}\AppData\Roaming\Trae CN` | Main data directory |
| User Dir | `c:\Users\{user}\.trae-cn` | User configuration |
| Install | `D:\Program Files\Trae CN` | Installation directory |

## Space Analysis Commands

### Analyze total space
```powershell
Get-ChildItem -Recurse -File | Measure-Object Length -Sum
```

### Find large directories
```powershell
Get-ChildItem -Directory | ForEach-Object { 
    $size = (Get-ChildItem $_.FullName -Recurse -File | Measure-Object Length -Sum).Sum
    [PSCustomObject]@{Folder=$_.Name; SizeMB=[math]::Round($size/1MB,2)}
} | Sort-Object SizeMB -Descending
```

### Find large files (>100MB)
```powershell
Get-ChildItem -Recurse -File | Where-Object { $_.Length -gt 100MB } | 
    Select-Object FullName, @{N='SizeMB';E={[math]::Round($_.Length/1MB,2)}} | 
    Sort-Object SizeMB -Descending
```

## Safe to Clean

1. **Logs** - `logs\` directory (old session logs)
2. **Cache** - `Cache\`, `GPUCache\`, `CachedData\`, `Crashpad\`
3. **Old Knowledge Projects** - `User\globalStorage\.ckg\storage\` (unused projects)

## Clean Commands

### Clean old logs (keep recent 3 sessions)
```powershell
Get-ChildItem -Directory | Where-Object { $_.Name -match '^\d{8}T\d{6}$' } | 
    Sort-Object Name -Descending | Select-Object -Skip 3 | 
    Remove-Item -Recurse -Force
```

### Clean cache directories
```powershell
Remove-Item "Cache\*" -Recurse -Force
Remove-Item "GPUCache\*" -Recurse -Force
Remove-Item "CachedData\*" -Recurse -Force
Remove-Item "Crashpad\*" -Recurse -Force
```

## Important Notes

1. **Do NOT delete** `database.db` files in ModularData - they contain chat history
2. **Do NOT delete** `IndexedDB` without user confirmation - it contains conversation data
3. Always analyze first before cleaning
4. Keep at least 3 recent log sessions for debugging

## Prevention Tips

1. Regularly clean old knowledge projects you no longer use
2. Set up periodic cache cleaning
3. Monitor space usage with the analysis commands
