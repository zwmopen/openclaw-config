# 时间盒子（Timeboxing）技能

## 触发条件

- 用户说「帮我规划今天的时间」
- 用户说「安排一下今天的任务」
- 用户说「提高效率」
- 用户说「时间不够用」
- 用户说「帮我做日程」

## 技能流程

### 1. 收集任务

询问用户：
- 今天要做什么？
- 有什么任务需要完成？
- 有什么固定的会议/预约？

### 2. 分类任务

| 类型 | 说明 | 时间盒类型 |
|------|------|-----------|
| 深度工作 | 需要高度专注 | 90分钟 |
| 常规工作 | 邮件、沟通、会议 | 30-60分钟 |
| 快速任务 | 15分钟内能完成 | 微型盒子 |

### 3. 分配时间盒

原则：
- 深度工作放在精力最好的时段（通常是上午）
- 常规工作放在下午
- 每个盒子之间留15分钟休息
- 每天留20%缓冲时间

### 4. 输出时间盒计划

```markdown
## 今日时间盒计划

### 上午（深度工作时段）
| 时间 | 任务 | 类型 |
|------|------|------|
| 09:00-10:30 | [任务1] | 深度工作 |
| 10:30-10:45 | 休息 | 缓冲 |
| 10:45-12:15 | [任务2] | 深度工作 |

### 下午（常规工作时段）
| 时间 | 任务 | 类型 |
|------|------|------|
| 14:00-15:00 | [任务3] | 常规工作 |
| 15:00-15:15 | 休息 | 缓冲 |
| 15:15-16:45 | [任务4] | 深度工作 |

### 晚上（快速任务时段）
| 时间 | 任务 | 类型 |
|------|------|------|
| 19:00-19:15 | [任务5] | 微型盒子 |
| 19:15-19:30 | [任务6] | 微型盒子 |

### 缓冲时间
- 16:45-17:15（30分钟）
```

### 5. 提醒规则

告诉用户：
1. 每个盒子只做一件事
2. 时间到就停，不管完成没有
3. 关掉所有通知，专注执行
4. 被打断就暂停计时，回来继续

## 快速规划模板

### 创作者日程

```
上午（深度创作）：
09:00-10:30 写脚本/拍视频
10:45-12:15 剪辑/修图

下午（运营工作）：
14:00-15:00 回复评论/私信
15:15-16:45 内容策划

晚上（快速任务）：
19:00-19:30 发布内容
```

### 自由职业日程

```
上午（核心工作）：
09:00-10:30 客户项目1
10:45-12:15 客户项目2

下午（沟通协作）：
14:00-15:00 客户会议
15:15-16:45 学习/提升

晚上（复盘规划）：
19:00-19:30 今日复盘
19:30-20:00 明日规划
```

## 注意事项

1. 不要排太满，留缓冲
2. 深度工作时段保护起来
3. 时间盒是工具，不是牢笼
4. 定期复盘，调整时间估算

## 完整案例：用户口述 → AI规划

用户口述：
> 我现在需要搞代运营，甲方是化妆培训学校...
> 第一件事：找对标账号，SEO关键词表、同行Excel...
> 第二件事：写简历、面试技巧文档...
> 预计3小时完成...

AI输出：
```
总时长：3小时（15:00–18:00）

任务1：小红书代运营对标+SEO（90min）
├── 时间盒1：15:00-15:20｜SEO关键词1.0
├── 时间盒2：15:20-15:50｜同行Excel表
├── 时间盒3：15:50-16:10｜笔记采集SOP
└── 时间盒4：16:10-16:30｜自检文档

任务2：简历+面试技巧（90min）
├── 时间盒5：16:30-17:15｜简历1.0
├── 时间盒6：17:15-17:45｜面试技巧文档
└── 时间盒7：17:45-18:00｜收尾整合
```
