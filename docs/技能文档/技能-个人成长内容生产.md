# 技能：个人成长内容生产

> OpenClaw 技能文件

---

## 技能说明

这个技能帮助用户把碎片想法变成有影响力的作品。

核心流程：口述采集 → 深化研究 → 创作输出 → 发布记录

---

## 触发词

### 采集类

| 触发词 | 动作 |
|--------|------|
| `口述：` | 语音转文字 + 纠错 + 结构化 + 存档 |
| `记录想法：` | 快速记录碎片想法 |
| `灵感：` | 快速记录 + 标记高优先级 |

### 深化类

| 触发词 | 动作 |
|--------|------|
| `深化：` | 深度研究 + 扩展分析 + 产出报告 |
| `研究：` | 多AI交叉研究 |
| `梳理：` | 结构化整理已有想法 |

### 创作类

| 触发词 | 动作 |
|--------|------|
| `写：` | 生成纯文字内容 |
| `写公众号：` | 生成公众号图文 |
| `写小红书：` | 生成小红书图文 |
| `写抖音：` | 生成抖音口述脚本 |

### 发布类

| 触发词 | 动作 |
|--------|------|
| `待发布：` | 标记为待发布 |
| `已发布：` | 记录发布数据 |

---

## 执行动作

### 动作1：口述采集

**触发**：用户说"口述：xxx"

**步骤**：
1. 接收用户的口述内容
2. 纠正错别字
3. 结构化整理
4. 提取核心观点
5. 标记待深化问题
6. 保存到 `00-口述采集/YYYY-MM-DD_主题关键词.md`
7. 回复用户：已记录 + 核心观点摘要

### 动作2：深化研究

**触发**：用户说"深化：xxx"

**步骤**：
1. 检索用户的知识库（5000+笔记）
2. 使用多个AI进行研究（豆包、GPT、DeepSeek）
3. 扩展分析
4. 产出深化报告
5. 保存到 `01-深化产出/深化_主题关键词_YYYY-MM-DD.md`
6. 回复用户：已深化 + 核心发现

### 动作3：创作输出

**触发**：用户说"写小红书：xxx" 等

**步骤**：
1. 调用用户的写作风格
2. 根据平台要求生成内容
3. 保存到 `02-待发布/平台_主题关键词_YYYY-MM-DD.md`
4. 回复用户：生成的内容 + 待发布

### 动作4：发布记录

**触发**：用户说"已发布：xxx"

**步骤**：
1. 移动文件到 `03-已发布/`
2. 记录发布数据
3. 更新数据统计
4. 回复用户：已记录

## 文件命名规范

### 口述采集
```
格式：YYYY-MM-DD_主题关键词.md
示例：2026-03-04_为什么学了道理还是过不好.md
```

### 深化产出
```
格式：深化_主题关键词_YYYY-MM-DD.md
示例：深化_道理与行动的差距_2026-03-04.md
```

### 待发布
```
格式：平台_主题关键词_YYYY-MM-DD.md
示例：小红书_道理与行动_2026-03-04.md
```

### 已发布
```
格式：平台_主题关键词_发布日期.md
示例：小红书_道理与行动_2026-03-05.md
```

## 注意事项

1. **快速响应**：用户说一句话，立即执行
2. **纠错优先**：口述内容先纠错，再整理
3. **风格一致**：创作时调用用户的写作风格
4. **数据追踪**：发布后记录数据，便于复盘
