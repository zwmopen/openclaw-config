# 系统安全检查

评估并加固运行OpenClaw的主机，然后根据用户定义的风险容忍度进行配置。

## 核心规则

- 建议使用高级模型运行此技能（如Opus 4.5, GPT 5.2+）
- 任何更改操作前必须获得明确批准
- 不要在未确认用户连接方式的情况下修改远程访问设置
- 优先使用可逆、分阶段的更改，并有回滚计划
- 永远不要声称OpenClaw会更改主机防火墙、SSH或OS更新，它不会
- 如果角色/身份未知，仅提供建议
- 每组用户选择必须编号，以便用户可以用单个数字回复

## 工作流程

### 1）建立上下文（只读）

尝试从环境推断1-5项，再询问。如需确认，优先使用简单的非技术性问题。

确定（按顺序）：

1. 操作系统和版本（Linux/macOS/Windows）、容器还是主机
2. 权限级别（root/admin还是用户）
3. 访问路径（本地控制台、SSH、RDP、tailnet）
4. 网络暴露（公网IP、反向代理、隧道）
5. OpenClaw网关状态和绑定地址
6. 备份系统和状态（如Time Machine、系统镜像、快照）
7. 部署上下文（本地mac应用、无头网关主机、远程网关、容器/CI）
8. 磁盘加密状态（FileVault/LUKS/BitLocker）
9. 操作系统自动安全更新状态
10. 使用模式（个人工作站还是无头/远程或其他）

如果必须询问，使用非技术性提示：

- "你用的是Mac、Windows电脑还是Linux？"
- "你是直接登录到这台机器，还是从另一台电脑连接？"
- "这台机器可以从公网访问，还是只在你的家里/网络内？"
- "你有启用备份吗（如Time Machine），备份是最新的吗？"
- "磁盘加密打开了吗（FileVault/BitLocker/LUKS）？"
- "自动安全更新启用了吗？"
- "你如何使用这台机器？"

### 2）运行OpenClaw安全审计（只读）

作为默认只读检查的一部分，运行：

```bash
openclaw security audit --deep
```

提供选项：

```bash
openclaw security audit        # 快速，不探测
openclaw security audit --json # 结构化输出
```

提供应用OpenClaw安全默认值：

```bash
openclaw security audit --fix
```

明确说明 `--fix` 只会收紧OpenClaw默认值和文件权限，不会更改主机防火墙、SSH或操作系统更新策略。

### 3）检查OpenClaw版本/更新状态（只读）

```bash
openclaw update status
```

报告当前渠道和是否有可用更新。

### 4）确定风险容忍度（系统上下文之后）

要求用户选择或确认风险姿态和任何需要开放的服​​务/端口。

建议配置文件（编号选择）：

1. **家庭/工作站平衡（最常见）**：防火墙开启合理默认值，远程访问限制在LAN或tailnet
2. **VPS加固**：默认拒绝入站防火墙，最小开放端口，仅密钥SSH，禁止root登录，自动安全更新
3. **开发者便捷**：允许更多本地服务，明确的暴露警告，仍然审计
4. **自定义**：用户定义的约束（服务、暴露、更新频率、访问方法）

### 5）生成修复计划

提供包括以下内容的计划：

- 目标配置
- 当前姿态摘要
- 与目标的差距
- 带精确命令的分步修复
- 访问保留策略和回滚
- 风险和潜在的锁定场景
- 最小权限说明
- 凭证卫生说明

在任何更改之前始终显示计划。

### 6）提供执行选项

1. 帮我执行（引导式，分步批准）
2. 只显示计划
3. 只修复关键问题
4. 导出命令供稍后执行

### 7）带确认执行

对于每一步：

- 显示精确命令
- 解释影响和回滚
- 确认访问仍然可用
- 在意外输出时停止并请求指导

### 8）验证并报告

重新检查：

- 防火墙状态
- 监听端口
- 远程访问仍然工作
- OpenClaw安全审计（重新运行）

提供最终姿态报告并注明任何延期项目。

## 必需确认（始终）

以下操作需要明确批准：

- 防火墙规则更改
- 开放/关闭端口
- SSH/RDP配置更改
- 安装/删除软件包
- 启用/禁用服务
- 用户/组修改
- 计划任务或启动持久性
- 更新策略更改
- 访问敏感文件或凭证

如果不确定，就问。

## 定期检查

OpenClaw安装或首次加固后，至少运行一次基线审计和版本检查：

```bash
openclaw security audit
openclaw security audit --deep
openclaw update status
```

建议进行持续监控。使用OpenClaw cron工具/CLI安排定期审计。

## OpenClaw命令准确性

仅使用支持的命令和标志：

```bash
openclaw security audit [--deep] [--fix] [--json]
openclaw status
openclaw status --deep
openclaw health --json
openclaw update status
openclaw cron add|list|runs|run
```

不要编造CLI标志或暗示OpenClaw执行主机防火墙/SSH策略。
