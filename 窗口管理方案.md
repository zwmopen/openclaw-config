## 创建窗口步骤

### 步骤1：创建飞书群聊

1. 打开飞书App
2. 点击右上角"+" → 创建群聊
3. 命名群聊（如"内容生产系统"、"代运营系统"）
4. 添加OpenClaw机器人到群聊

### 步骤2：告知窗口用途

在群聊中发送消息，告诉OpenClaw这个窗口的用途：

**示例**：
```
这个窗口是内容生产系统窗口，用于：
1. 内容创作
2. 选题管理
3. 素材整理
```

### 步骤3：OpenClaw自动加载配置

OpenClaw收到消息后，会：
1. 识别这是新窗口
2. 记录窗口用途
3. 从 `MEMORY.md` 加载相关配置
4. 从 `EVOLUTION.md` 加载任务状态
5. 告诉你准备就绪

## 窗口记忆机制

### 记忆文件

| 文件 | 用途 | 内容 |
|------|------|------|
| `MEMORY.md` | 长期记忆 | 用户信息、触发词、系统配置、重要发现 |
| `EVOLUTION.md` | 进化计划 | 进化阶段、任务状态、每日汇报 |
| `HEARTBEAT.md` | 心跳配置 | 定时任务、触发词、技能识别 |
| `状态快照-YYYY-MM-DD.md` | 状态快照 | 当前窗口、任务状态、配置备份 |

### 记忆恢复流程

1. **新窗口创建**：OpenClaw检测到新会话
2. **识别窗口用途**：用户告诉OpenClaw这个窗口是做什么的
3. **加载记忆**：OpenClaw读取 `MEMORY.md`、`EVOLUTION.md` 等文件
4. **恢复状态**：OpenClaw从状态快照恢复任务状态
5. **继续工作**：OpenClaw继续之前的任务

## 窗口切换规则

### 自动识别

OpenClaw会根据消息内容自动识别窗口用途：

**识别规则**：
- 包含"进化"、"状态"、"配置" → 主窗口
- 包含"口述"、"写小红书"、"写公众号" → 内容生产系统窗口
- 包含"收集爆款"、"对标"、"代运营" → 代运营系统窗口
- 包含"梳理笔记"、"素材库"、"方法论" → 个人知识库梳理窗口

### 手动告知

如果自动识别不准确，可以手动告知：

**示例**：
```
切换到内容生产系统窗口
这个窗口用于内容创作
```

## 窗口状态同步

### 跨窗口共享

以下信息在所有窗口共享：
- `MEMORY.md` - 长期记忆
- `EVOLUTION.md` - 进化计划
- `HEARTBEAT.md` - 心跳配置
- `USER.md` - 用户信息
- `SOUL.md` - 灵魂文件

### 窗口独立

以下信息在每个窗口独立：
- 当前任务
- 任务进度
- 会话上下文

### 同步机制

1. **实时同步**：修改共享文件后，所有窗口立即生效
2. **定期同步**：每小时同步一次任务状态
3. **手动同步**：用户说"同步状态"时立即同步

## 恢复方案

### 场景1：某个窗口丢失

**步骤**：
1. 创建新的飞书群聊
2. 添加OpenClaw机器人
3. 告知窗口用途：`这个窗口是[窗口名称]窗口，用于[用途]`
4. OpenClaw读取状态快照恢复记忆

### 场景2：所有窗口丢失

**步骤**：
1. 重新配置飞书Bot
2. 创建新的飞书群聊
3. 添加OpenClaw机器人
4. 告知主窗口用途：`这个窗口是主窗口，用于自我迭代和系统管理`
5. OpenClaw读取所有记忆文件恢复记忆
6. 逐个创建其他窗口

### 场景3：完全重置

**步骤**：
1. 备份以下文件：
   - `AGENTS.md`
   - `SOUL.md`
   - `USER.md`
   - `MEMORY.md`
   - `EVOLUTION.md`
   - `HEARTBEAT.md`
   - `状态快照-YYYY-MM-DD.md`
   - `.openclaw/openclaw.json`
   - `.openclaw/paired.json`
   - `安全配置/` 整个文件夹
   - `技能库/` 整个文件夹
2. 重新部署OpenClaw
3. 恢复备份文件
4. 重启OpenClaw服务
5. 创建窗口并告知用途
6. OpenClaw恢复记忆

## 窗口管理最佳实践

### 建议

1. **明确分工**：每个窗口只负责一类任务，避免混乱
2. **定期同步**：每小时同步一次状态，防止信息不一致
3. **备份记忆**：每天备份一次 `MEMORY.md` 和 `EVOLUTION.md`
4. **创建快照**：每周创建一次状态快照
5. **测试恢复**：每月测试一次恢复流程，确保可用

### 避免

1. **避免混用**：不要在一个窗口处理多种任务
2. **避免遗忘**：定期检查各窗口状态
3. **避免丢失**：备份重要文件
4. **避免冲突**：同一时间不要在多个窗口处理同一任务
