## 恢复方案

### 场景1：飞书API突然无法使用

**步骤**：
1. 检查飞书Bot配置（`安全配置/飞书/飞书机器人配置.md`）
2. 登录飞书开放平台，检查API配额
3. 如果配额用完，申请新的Bot或等待下月重置
4. 如果Bot被禁用，创建新的Bot并更新配置
5. 重启OpenClaw服务

### 场景2：创建新的飞书群聊

**步骤**：
1. 创建新的飞书群聊
2. 添加OpenClaw机器人到群聊
3. 在群聊中发送消息，OpenClaw会自动创建新的会话
4. 告诉OpenClaw你的窗口分工（如"这个窗口是代运营系统窗口"）
5. OpenClaw会读取此状态快照文件，恢复记忆

### 场景3：切换到其他应用

**步骤**：
1. 配置新的应用（如微信、Telegram等）
2. 更新 `openclaw.json` 中的通道配置
3. 重启OpenClaw服务
4. 在新应用中发送消息，OpenClaw会自动创建新的会话
5. 告诉OpenClaw你的窗口分工
6. OpenClaw会读取此状态快照文件，恢复记忆

### 窗口划分建议

**方案A：按功能划分**
1. **主窗口（自我迭代）**：进化、系统管理、配置修改
2. **内容生产系统窗口**：内容创作、选题管理、素材整理
3. **代运营系统窗口**：代运营工作、对标收集、爆款分析
4. **个人知识库梳理窗口**：笔记整理、知识管理、素材库填充

**方案B：按优先级划分**
1. **高优先级窗口**：紧急任务、重要决策
2. **中优先级窗口**：日常工作、内容生产
3. **低优先级窗口**：知识管理、长期规划

**方案C：按时间划分**
1. **上午窗口**：内容创作、代运营工作
2. **下午窗口**：知识管理、系统优化
3. **晚上窗口**：复盘、规划

### 窗口记忆机制

每个窗口创建后，OpenClaw会：
1. 记录窗口用途（用户告知）
2. 加载相关配置（从MEMORY.md读取）
3. 恢复相关任务状态（从EVOLUTION.md读取）
4. 继续之前的任务

**示例**：
```
用户：这个窗口是代运营系统窗口
OpenClaw：收到！这是代运营系统窗口，我会：
1. 处理代运营相关工作
2. 收集爆款笔记和对标账号
3. 分析数据并生成报告
4. 从上次的状态继续工作
```
