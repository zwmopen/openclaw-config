# 笔记梳理 Skill

## 触发条件

- 用户说「梳理笔记」
- 用户说「整理笔记」
- 用户说「批量处理笔记」
- 用户说「把这篇笔记梳理一下」

## 梳理规则

### 文件处理

1. 扫描 Obsidian 笔记库目录：`D:\Program Files\Obsidian\zwm\.zwm\`
2. 跳过已梳理的文件（文件名包含「已梳理」）
3. 跳过隐藏目录（.obsidian, .trae, .git等）
4. 跳过配置文件（AGENTS.md, SOUL.md, USER.md等）

### 输出格式

创建新文件：`原文件名（已梳理）.md`

```markdown
# [原标题]（已梳理）

## 一句话总结
[核心观点，不超过20字]

## 核心认知
[提取3-5个核心认知，每个2-3句话]
- **认知1**：说明
- **认知2**：说明
...

## 实践要点
[可执行的行动点]

### 想复刻？你需要：
1. 步骤1
2. 步骤2
...

### 避坑：
- 坑1
- 坑2

## 自媒体选题
| 方向 | 标题参考 |
|------|---------|
| 方向1 | 标题1 |
| 方向2 | 标题2 |

## 内容来源
- 原始文件：[原文件名]
- 梳理时间：[日期]
- 核心领域：#[标签1] #[标签2]
```

### 梳理原则

1. **不要太官方** - 自然、直接、能落地
2. **提取核心认知** - 真正有价值的洞察
3. **给出实践要点** - 可执行的行动点
4. **生成自媒体选题** - 可变现的内容方向
5. **保留原文核心** - 不要过度解读

## 工作流程

### 单篇梳理

1. 读取原始笔记
2. 提取核心观点
3. 按格式整理
4. 创建新文件保存

### 批量梳理

1. 扫描目录，找出未梳理的文件
2. 每次处理10-20篇
3. 定期报告进度
4. 记录已处理的文件，避免重复

## 进度报告格式

```
📊 笔记梳理进度

- 已处理：X/5666
- 本次处理：X篇
- 剩余：XXXX篇
- 预估时间：约X小时

已梳理文件：
- 文件1.md ✅
- 文件2.md ✅
...
```

## 注意事项

1. 控制API调用频率，避免超限
2. 每处理20篇报告一次进度
3. 遇到内容太短的笔记，跳过或合并
4. 遇到图片/视频笔记，提取文字描述
